# BUG-003 — Campo Postal Code acepta valores no numéricos

## Descripción
El campo Postal Code del formulario de checkout no valida que el valor 
ingresado sea numérico, permitiendo completar la compra con texto arbitrario.

## Pasos para reproducir
1. Ingresar a https://www.saucedemo.com
2. Login con standard_user / secret_sauce
3. Agregar cualquier producto al carrito
4. Ir al carrito y click en "Checkout"
5. Completar First Name y Last Name con cualquier valor
6. Ingresar texto (ej: "abcde") en el campo Postal Code
7. Click en "Continue"

## Resultado esperado
El sistema debería mostrar un error indicando que el Postal Code debe 
contener solo valores numéricos.

## Resultado obtenido
La compra avanza sin ningún error y se completa exitosamente.

## Impacto
Datos inválidos podrían almacenarse en la base de datos, causando problemas 
en procesos logísticos o de envío que dependan del código postal.

## Nota
En algunos países el código postal incluye letras (ej: Canada, UK). 
Si la app opera en esos mercados, este comportamiento podría ser intencional. 
Se recomienda clarificar el requisito con el equipo de producto.
